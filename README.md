# SecondBrain - VS Code 第二大脑系统

这是一个基于 VS Code 构建的 PARA + Zettelkasten 混合知识管理系统，集成了强大的网页剪辑功能。

## 🎯 系统特色

✅ **双系统融合**: PARA 项目管理 + Zettelkasten 知识网络  
✅ **网页剪辑集成**: 一键保存网页内容到统一收集箱  
✅ **跨平台兼容**: 完全兼容 Obsidian、Notion 等主流平台  
✅ **模板驱动**: 丰富的代码片段和自动化模板  
✅ **版本控制**: Git 集成，追踪知识演化历程  

## 🗂️ 目录结构

```
SecondBrain/
├── PARA/                    # PARA 方法论系统
│   ├── 01_Projects/        # 项目：有明确结果和截止日期的工作
│   ├── 02_Areas/           # 领域：需要持续维护的生活领域
│   ├── 03_Resources/       # 资源：未来可能有用的参考材料
│   └── 04_Archive/         # 归档：来自其他三个类别的不活跃项目
├── ZK/                     # Zettelkasten 卢曼笔记法
│   ├── 01_Fleeting/        # 速记：临时想法和待整理内容
│   ├── 02_Literature/      # 文献笔记：书籍、文章等的读书笔记
│   └── 03_Permanent/       # 永久笔记：经过思考的知识卡片
├── Inbox/                  # 统一收集箱
│   ├── web-clips/          # 网页剪辑内容（自动分类）
│   ├── quick-notes/        # 快速记录
│   └── daily/              # 每日收集
└── .vscode/                # VS Code 配置
    ├── settings.json       # 工作区设置
    ├── tasks.json          # 自动化任务
    └── markdown.code-snippets  # 模板片段
```

## 🚀 10分钟快速开始

### 第一步：安装推荐扩展 (2分钟)

VS Code 会自动提示安装推荐扩展，包括：
- **Markdown All in One** - Markdown 编辑增强
- **Foam** - 知识网络可视化和双向链接
- **Paste Image** - 快速粘贴截图
- **Todo Tree** - 任务管理
- **Draw.io Integration** - 绘图工具

### 第二步：配置网页剪辑 (3分钟)

安装 **Obsidian Web Clipper** 浏览器扩展：
- **Chrome/Edge**: 在应用商店搜索 "Obsidian Web Clipper"
- **Firefox**: 在扩展商店搜索 "Obsidian Web Clipper" 
- **Safari**: 支持 Chrome 扩展或直接下载

**配置步骤**：
1. 安装扩展后点击浏览器工具栏的剪辑图标
2. 选择 VS Code 工作区路径：`F:\github\SecondBrain`
3. 设置默认保存位置：`Inbox/web-clips/` 文件夹
4. 选择合适的剪辑模板（文章、参考资料等）

### 第三步：创建第一个笔记 (3分钟)

在 Markdown 文件中输入代码片段：

**PARA 系统快速模板**：
- `/para-project` + Tab → 创建项目模板
- `/para-project-dev` + Tab → 软件开发项目
- `/para-project-ai` + Tab → AI/大模型项目
- `/para-area` + Tab → 创建领域模板
- `/para-resource` + Tab → 创建资源模板

**ZK 系统快速模板**：
- `/zk-fleeting` + Tab → 速记笔记
- `/zk-literature` + Tab → 文献笔记
- `/zk-literature-tech` + Tab → 技术学习笔记
- `/zk-permanent` + Tab → 永久笔记
- `/zk-daily` + Tab → 日记模板

### 第四步：使用核心功能 (2分钟)

**快捷键**：
- `Ctrl+Alt+D` → 打开今日笔记
- `Ctrl+Alt+G` → 显示知识图谱
- `Ctrl+Alt+R` → 打开随机笔记
- `Ctrl+Alt+P` → 快速打开文件

**快捷任务** (按 `Ctrl+Shift+P` 输入 "Tasks: Run Task")：
- **🔍 创建日记** - 自动创建今天的日记文件
- **🚀 快速创建PARA项目** - 交互式项目创建
- **💾 备份到Git** - 快速提交所有更改

## 📝 核心使用方法

### 🌐 网页剪辑工作流

**1. 即时剪辑**：
- 浏览网页时点击 Obsidian Web Clipper 图标
- 选择剪辑类型：完整文章、高亮内容、或自定义选择
- 自动保存到 `Inbox/web-clips/` 文件夹

**2. 内容分类策略**：
- **技术文章** → 移至 `PARA/03_Resources/` 或转为 `ZK/02_Literature/`
- **项目参考** → 关联到相关项目文件夹 `PARA/01_Projects/`
- **学习资源** → 发展为 `ZK/01_Fleeting/` 或 `ZK/03_Permanent/` 笔记
- **新闻资讯** → 快速浏览后归档或删除

**3. 知识转化流程**：
```
网页剪辑 → 初步整理 → 提取关键信息 → 创建永久笔记 → 建立双向链接
```

### 📋 PARA 项目管理工作流

1. **项目 (Projects)**: 为每个项目创建文件夹，包含 `index.md` 主页
2. **领域 (Areas)**: 按生活/工作领域组织，如健康、财务、技能等
3. **资源 (Resources)**: 存放参考资料、模板、清单和网页剪辑
4. **归档 (Archive)**: 使用任务自动归档已完成项目

### 🧠 Zettelkasten 知识网络工作流

1. **收集想法**: 在 `Inbox/` 和 `01_Fleeting/` 中快速记录想法和剪辑内容
2. **整理文献**: 将读书笔记和网页剪辑整理到 `02_Literature/`
3. **提炼知识**: 在 `03_Permanent/` 中创建原子化知识卡片
4. **建立连接**: 使用 `[[链接]]` 语法连接相关笔记

### 🔗 链接语法和跨系统连接

- **PARA 内部链接**: `[文档名](../Resources/文档.md)`
- **Zettelkasten 链接**: `[[UID-标题]]` 或 `[[标题]]`
- **Inbox 内容链接**: `[[Inbox/web-clips/文章标题]]`
- **网页剪辑引用**: `[[Clippings/Articles/文章标题]]`
- **跨系统连接**: `[[PARA/01_Projects/项目名]] → [[ZK/03_Permanent/相关概念]]`

## 📱 移动端和跨平台支持

1. **Git 版本控制**: 所有内容同步到 GitHub/GitLab

### 平台迁移兼容性

- **标准 Markdown**: 任何支持 Markdown 的平台都能打开

## 🎯 最佳实践和管理策略

### 📊 Inbox 管理策略

**剪辑内容分类**：
- **技术文章**: 开发相关的技术博客、文档
- **学习资源**: 教程、课程、最佳实践
- **项目参考**: 设计案例、解决方案、工具推荐
- **新闻资讯**: 行业动态、技术趋势

**定期整理规则**：
- **每日**: 快速浏览新剪辑，标记重要内容
- **每周**: 将有价值剪辑移动到对应的 PARA/ZK 位置
- **每月**: 清理过期或价值不高的剪辑
- **每季**: 回顾剪辑转化效果，优化收集策略

### 🔄 高效工作流实践

**网页剪辑集成流程**：
1. **即时剪辑** - 浏览时遇到有价值内容立即剪辑
2. **每日整理** - 查看当日剪辑，进行初步分类
3. **周度转化** - 将有价值剪辑转化为正式笔记
4. **月度清理** - 清理过期或价值不高的剪辑
5. **季度归档** - 将重要剪辑归档到对应 PARA/ZK 位置

**跨系统协作原则**：
- 在永久笔记中链接相关的项目和资源
- 在项目中引用知识卡片支持决策
- 用标签系统连接两个体系的内容
- 建立连接时主动寻找与现有笔记的联系

## 🆘 常见问题

**Q: 两套系统会不会太复杂？**  
A: 可以从其中一套开始，比如先用 PARA 管理项目，再逐步加入 ZK。系统设计支持渐进式采用。

**Q: 网页剪辑会不会造成信息过载？**  
A: 建议采用"质量优先"策略，精选高质量内容，并建立定期清理机制。

**Q: 如何在两个系统间建立有效连接？**  
A: 使用双向链接 `[[]]` 在项目中引用知识卡片，在笔记中链接相关项目。关键是建立主题标签系统。

**Q: Git 备份有什么具体好处？**  
A: 版本控制让你可以追踪知识的演化历程，永不丢失重要内容，还支持多设备同步。

**Q: 移动端如何使用这个系统？**  
A: 推荐使用 Obsidian Mobile 查看和编辑，通过 Git 实现跨设备同步。移动端也支持网页剪辑。

- **信息获取效率提升**: 一键保存网页内容，告别书签混乱
